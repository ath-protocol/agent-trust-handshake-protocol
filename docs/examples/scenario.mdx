This example walks through a complete ATH flow where a travel agent checks flight confirmation emails on behalf of a user.

## Participants

- **User**: Alice, has an email account
- **Agent**: TravelBot (`https://travel-agent.example.com/.well-known/agent.json`)
- **Service**: An email service
- **ATH Implementor**: Gateway or native service

## Phase A: Agent Registration (One-Time)

1. TravelBot developer registers with the ATH implementor:
   - Submits agent identity, attestation JWT, and requests `mail:read` scope

2. The implementor reviews and approves TravelBot for `mail:read`

3. TravelBot receives `client_id`

## Phase B: Runtime (Alice Wants to Check Emails)

**Step 1**: Alice tells TravelBot: "Find confirmation emails for my Tokyo trip"

**Step 2**: TravelBot sends authorization request:

```
POST /ath/authorize
{
  "client_id": "ath_travelbot_001",
  "provider_id": "example-mail",
  "scopes": ["mail:read"]
}
```

**Step 3**: Implementor verifies:
- TravelBot is registered and approved (**app-side auth** ✓)
- TravelBot is approved for `mail:read` ✓

**Step 4**: Implementor initiates OAuth flow → email service consent page

**Step 5**: Alice sees the consent page:

> "TravelBot wants to view your email messages"

**Step 6**: Alice approves (**user-side auth** ✓)

**Step 7**: Implementor receives OAuth token, computes scope intersection:

```
Agent approved:  mail:read  ✓
User consented:  mail:read  ✓
Effective scope: mail:read
```

**Step 8**: Implementor issues ATH access token to TravelBot, bound to:
- Agent: TravelBot
- User: Alice
- Provider: example-mail
- Scopes: mail:read

**Step 9**: TravelBot calls email API:

```
GET /ath/proxy/example-mail/v1/messages?q=Tokyo+trip
Authorization: Bearer ath_tk_xxxxxxxx
```

**Step 10**: Implementor validates token, serves/proxies the request, returns results

**Step 11**: TravelBot summarizes the confirmation emails for Alice

## What If TravelBot Tries to Send Email?

If TravelBot attempts to call a send endpoint:

```
POST /ath/proxy/example-mail/v1/messages/send
Authorization: Bearer ath_tk_xxxxxxxx
```

The implementor **rejects** the request because:
- The ATH token is bound to `mail:read` only
- `mail:send` was never in the effective scope
- The trusted handshake ensures the agent cannot exceed its approved capabilities
