本协议采用8步流程实现端到端可信授权，与项目README中的描述完全对齐。
## 8步可信握手流程
```
用户          智能体        ATH实现方          服务提供方
 │             │                  │                │
 │  "执行操作" │                  │                │
 ├────────────►│                  │                │
 │             │                  │                │
 │  1. 请求访问权限            │                │
 │             ├─────────────────►│                │
 │             │                  │                │
 │             │  2. 验证智能体身份与审批状态  │
 │             │                  │                │
 │             │  3. 启动OAuth授权流程          │
 │             │                  ├───────────────►│
 │             │                  │                │
 │  4. 显示OAuth授权页面         │                │
 │◄───────────────────────────────────────────────┤
 │                                │                │
 │  5. 用户同意授权              │                │
 ├───────────────────────────────────────────────►│
 │             │                  │                │
 │             │  6. 权限交集计算与令牌绑定  │
 │             │                  │                │
 │             │  7. 签发ATH访问令牌       │
 │             │◄─────────────────┤                │
 │             │                  │                │
 │             │  8. 认证API调用与结果返回  │
 │             ├─────────────────►│                │
 │             │                  ├───────────────►│
 │             │                  │◄───────────────┤
 │  最终结果   │                  │                │
 │◄────────────┤                  │                │
```
<Note>
在网关模式下，"ATH实现方"是网关，"服务提供方"是上游服务。在原生模式下，"ATH实现方"和"服务提供方"是同一个实体。
</Note>
## 权限交集（第6步）
任何访问令牌的有效权限是以下三个集合的**交集**：
```
有效权限 = 智能体已审批权限 ∩ 用户同意权限 ∩ 当前请求权限
```
详见[权限交集](/zh/specification/0.1/client/scope-intersection)。
