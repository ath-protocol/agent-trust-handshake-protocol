## Agent Identity (Agent_ID)

Every agent MUST have a unique, verifiable identity.

**Format**: URI-based identifier following the pattern:

```
https://<agent-domain>/.well-known/agent.json
```

**Examples**:
```
https://travel-agent.example.com/.well-known/agent.json
https://coding-assistant.example.com/.well-known/agent.json
```

## Agent Identity Document

The agent MUST publish a JSON document at their `agent_id` URI:

```json
{
  "ath_version": "0.1",
  "agent_id": "https://travel-agent.example.com/.well-known/agent.json",
  "name": "TravelBot",
  "developer": {
    "name": "Example Corp",
    "id": "dev-example-12345",
    "contact": "security@example.com"
  },
  "capabilities": ["flight-search", "hotel-booking", "itinerary-planning"],
  "public_key": "<JWK or PEM for agent attestation verification>"
}
```

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `ath_version` | string | Protocol version (e.g., `"0.1"`) |
| `agent_id` | string | Canonical URI for this agent |
| `name` | string | Human-readable agent name |
| `developer` | object | Developer information |
| `developer.name` | string | Developer or organization name |
| `developer.id` | string | Developer identifier |
| `public_key` | string/object | JWK or PEM for attestation verification |

### Optional Fields

| Field | Type | Description |
|-------|------|-------------|
| `developer.contact` | string | Security contact email |
| `capabilities` | string[] | List of agent capabilities |

## Service Identity (App_ID)

Services are identified by their existing identifiers:
- **Client-side apps**: Package name (e.g., `com.example.mail`)
- **Server-side services**: URI (e.g., `https://api.example.com`)

## Agent Attestation

Agents prove their identity using a signed JWT (Agent Attestation Token).

### JWT Header

```json
{
  "alg": "ES256",
  "typ": "JWT",
  "kid": "<key-id>"
}
```

### JWT Payload

```json
{
  "iss": "https://travel-agent.example.com",
  "sub": "<agent_id>",
  "aud": "<target-service-or-gateway>",
  "iat": 1720000000,
  "exp": 1720003600,
  "capabilities": ["flight-search"]
}
```

### Required Claims

| Claim | Description |
|-------|-------------|
| `iss` | Issuer — the agent's domain |
| `sub` | Subject — the full `agent_id` URI |
| `aud` | Audience — the target service or gateway URL |
| `iat` | Issued at — timestamp |
| `exp` | Expiration — MUST be present |

### Verification

The verifier (gateway or service):
1. Fetches the agent's identity document from the `agent_id` URI
2. Extracts the public key
3. Verifies the JWT signature using the public key
4. Validates `exp` (not expired) and `aud` (matches the verifier's URL)
5. Implementors MUST reject expired or audience-mismatched attestations
