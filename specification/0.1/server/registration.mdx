## Agent Registration (Phase A)

**`POST /ath/agents/register`**

Registers a new agent with the ATH implementor. The implementor verifies the agent's identity and evaluates the requested capabilities against its approval policy.

## Request

```json
{
  "agent_id": "https://travel-agent.example.com/.well-known/agent.json",
  "agent_attestation": "<signed JWT>",
  "developer": {
    "name": "Example Corp",
    "id": "dev-example-12345"
  },
  "requested_providers": [
    {
      "provider_id": "example-mail",
      "scopes": ["mail:read", "mail:send"]
    }
  ],
  "purpose": "Travel planning assistant",
  "redirect_uris": ["https://travel-agent.example.com/callback"]
}
```

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `agent_id` | string | Yes | The agent's canonical URI |
| `agent_attestation` | string | Yes | Signed JWT proving agent identity |
| `developer` | object | Yes | Developer information |
| `developer.name` | string | Yes | Developer or organization name |
| `developer.id` | string | Yes | Developer identifier |
| `requested_providers` | array | Yes | Providers and scopes to request |
| `requested_providers[].provider_id` | string | Yes | Provider identifier |
| `requested_providers[].scopes` | string[] | Yes | Requested scopes for this provider |
| `purpose` | string | No | Human-readable description of agent's purpose |
| `redirect_uris` | string[] | No | OAuth callback URIs |

## Response (Success)

```json
{
  "client_id": "ath_travelbot_001",
  "client_secret": "ath_secret_xxxxx",
  "agent_status": "approved",
  "approved_providers": [
    {
      "provider_id": "example-mail",
      "approved_scopes": ["mail:read"],
      "denied_scopes": ["mail:send"],
      "denial_reason": "Send capability requires additional review"
    }
  ],
  "approval_expires": "2027-01-01T00:00:00Z"
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `client_id` | string | Unique identifier for this registration |
| `client_secret` | string | Secret for authenticating token exchange requests |
| `agent_status` | string | `"approved"`, `"pending"`, or `"denied"` |
| `approved_providers` | array | Per-provider approval results |
| `approved_providers[].approved_scopes` | string[] | Scopes that were approved |
| `approved_providers[].denied_scopes` | string[] | Scopes that were denied |
| `approved_providers[].denial_reason` | string | Reason for denial (if any) |
| `approval_expires` | string | ISO 8601 expiration timestamp |

## Behavior

- The implementor MAY approve a **subset** of requested capabilities
- The agent MUST respect the approved scope limits
- Duplicate registrations for the same `agent_id` SHOULD return a `409 Conflict`
