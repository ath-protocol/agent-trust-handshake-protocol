## Agent Registration (Phase A)

**`POST /ath/agents/register`**

Register a new agent with the ATH implementer. The implementer verifies the agent's identity and evaluates the requested capabilities according to its approval policy.

## Request

```json
{
  "agent_id": "https://travel-agent.example.com/.well-known/agent.json",
  "agent_attestation": "<signed JWT>",
  "developer": {
    "name": "Example Corp",
    "id": "dev-example-12345"
  },
  "requested_providers": [
    {
      "provider_id": "example-mail",
      "scopes": ["mail:read", "mail:send"]
    }
  ],
  "purpose": "Travel planning assistant",
  "redirect_uris": ["https://travel-agent.example.com/callback"]
}
```

### Request Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `agent_id` | string | Yes | The agent's canonical URI |
| `agent_attestation` | string | Yes | A signed JWT attesting to the agent's identity |
| `developer` | object | Yes | Developer information |
| `developer.name` | string | Yes | Developer or organization name |
| `developer.id` | string | Yes | Developer identifier |
| `requested_providers` | array | Yes | Requested service providers and scopes |
| `requested_providers[].provider_id` | string | Yes | Service provider identifier |
| `requested_providers[].scopes` | string[] | Yes | Scopes requested for this service provider |
| `purpose` | string | No | Human-readable description of the agent's purpose |
| `redirect_uris` | string[] | Yes | OAuth callback URIs (validated by exact-match during authorization) |

## Response (Success)

```json
{
  "client_id": "ath_travelbot_001",
  "client_secret": "ath_secret_xxxxx",
  "agent_status": "approved",
  "approved_providers": [
    {
      "provider_id": "example-mail",
      "approved_scopes": ["mail:read"],
      "denied_scopes": ["mail:send"],
      "denial_reason": "Send capability requires additional review"
    }
  ],
  "approval_expires": "2027-01-01T00:00:00Z"
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `client_id` | string | Unique identifier for this registration |
| `client_secret` | string | Secret used to authenticate token exchange requests |
| `agent_status` | string | `"approved"`, `"pending"`, or `"denied"` |
| `approved_providers` | array | Approval results for each service provider |
| `approved_providers[].approved_scopes` | string[] | Approved scopes |
| `approved_providers[].denied_scopes` | string[] | Denied scopes |
| `approved_providers[].denial_reason` | string | Reason for denial (if any) |
| `approval_expires` | string | Expiration timestamp in ISO 8601 format |

## Behavior

- The implementer MAY approve a **subset** of the requested capabilities
- The agent MUST respect the approved scope restrictions
- Duplicate registrations for the same `agent_id` SHOULD return `409 Conflict`
