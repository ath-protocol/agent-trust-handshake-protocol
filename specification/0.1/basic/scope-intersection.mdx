## Definition

The effective scope of any ATH access token MUST be computed as:

```
Effective Scope = Agent Approved Scopes ∩ User Consented Scopes ∩ Requested Scopes
```

Where:
- **Agent Approved Scopes**: The set of scopes the ATH implementor approved for the agent during Phase A registration
- **User Consented Scopes**: The set of scopes the user granted during the Phase B OAuth consent flow
- **Requested Scopes**: The set of scopes the agent requested in the authorization request

## Example

```
Agent approved scopes:    mail:read, mail:send
User consented scopes:    mail:read, mail:send, mail:delete
Agent requested scopes:   mail:read
────────────────────────────────────────────────
Effective scope:          mail:read
```

## Requirements

1. Implementors MUST compute scope intersection before issuing any ATH access token
2. The token response MUST include the `scope_intersection` breakdown
3. Implementors MUST NOT issue tokens with scopes outside the computed intersection
4. If the intersection is empty, the implementor MUST NOT issue a token

## Token Response Format

```json
{
  "access_token": "ath_tk_xxxxxxxx",
  "token_type": "Bearer",
  "expires_in": 3600,
  "effective_scopes": ["mail:read"],
  "provider_id": "example-mail",
  "agent_id": "https://travel-agent.example.com/.well-known/agent.json",
  "scope_intersection": {
    "agent_approved": ["mail:read"],
    "user_consented": ["mail:read", "mail:send"],
    "effective": ["mail:read"]
  }
}
```

The `scope_intersection` field provides full transparency about how the effective scopes were derived.
