# ATH协议握手流程规范
## 概述
ATH协议握手过程包含3个核心阶段和12个标准步骤，实现异构AI智能体之间的安全信任建立。
## 3个核心阶段
### 第一阶段：身份验证（6步）
通过去中心化身份注册中心进行双向身份验证，确保交互双方都是合法主体。
### 第二阶段：权限协商（3步）
基于OAuth 2.0扩展框架进行细粒度权限协商，实现最小权限授权原则。
### 第三阶段：会话建立（3步）
采用TLS 1.3协议建立端到端加密通信会话。
## 完整12步握手流程
| 步骤 | 阶段 | 描述 |
|------|------|------|
| 1 | 身份验证 | 智能体A向智能体B发送握手请求，包含去中心化身份标识DID和能力清单 |
| 2 | 身份验证 | 智能体B将智能体A的身份信息发送到身份注册中心进行有效性验证 |
| 3 | 身份验证 | 身份注册中心向智能体B返回身份验证结果 |
| 4 | 身份验证 | 智能体B向智能体A返回握手响应，包含自身的DID和能力清单 |
| 5 | 身份验证 | 智能体A将智能体B的身份信息发送到身份注册中心进行有效性验证 |
| 6 | 身份验证 | 身份注册中心向智能体A返回身份验证结果 |
| 7 | 权限协商 | 智能体A向智能体B发送权限请求清单，包含请求访问的接口和数据范围 |
| 8 | 权限协商 | 智能体B向智能体A返回权限审批结果（同意/拒绝/部分同意），包含有效时间 |
| 9 | 权限协商 | 智能体A向智能体B发送权限范围确认 |
| 10 | 会话建立 | 智能体A向智能体B发送会话密钥协商请求 |
| 11 | 会话建立 | 智能体B向智能体A返回会话密钥协商响应 |
| 12 | 会话建立 | 双方建立加密通信会话 |
## 核心特性
1. **去中心化身份**：采用DID作为唯一身份标识，无单点故障问题
2. **双向验证**：双方都通过独立的注册中心验证对方身份
3. **细粒度权限**：支持接口级、数据级、时间级的权限控制
4. **端到端加密**：基于TLS 1.3加密，确保通信安全
5. **可审计**：所有握手过程都有记录，可审计可追溯
