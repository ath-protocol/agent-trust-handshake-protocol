# 握手流程
ATH协议的核心是8步可信握手流程，在客户端和服务端之间建立双向可信的通信通道。
## 8步握手流程概览
```mermaid
sequenceDiagram
    participant C as 客户端
    participant S as 服务端
    
    Note over C,S: 8步握手流程：
1. 客户端身份宣告 (Client Identity Announcement)
2. 服务端身份回应 (Server Identity Response)
3. 作用域请求 (Scope Request)
4. 作用域协商 (Scope Negotiation)
5. 身份验证 (Identity Verification)
6. 凭证颁发 (Credential Issuance)
7. 凭证确认 (Credential Confirmation)
8. 握手完成 (Handshake Complete)

## 详细步骤说明
### 8步握手流程：
1. 客户端身份宣告 (Client Identity Announcement)
2. 服务端身份回应 (Server Identity Response)
3. 作用域请求 (Scope Request)
4. 作用域协商 (Scope Negotiation)
5. 身份验证 (Identity Verification)
6. 凭证颁发 (Credential Issuance)
7. 凭证确认 (Credential Confirmation)
8. 握手完成 (Handshake Complete)

## 安全特性
- **双向身份认证**：通过非对称加密确保双方身份真实可信
- **最小权限原则**：服务端只授予客户端必要的访问权限
- **短期凭证**：访问凭证有效期短，降低凭证泄露风险
- **不可抵赖性**：所有交互都有签名，可审计可追溯
